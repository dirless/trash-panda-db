GitHub Documentation
🗃️

Embedded or Replicated

Use it in-process via the crystal-db driver, or deploy a fault-tolerant 3-node cluster.

⚡

No Dependencies

Pure Crystal. No C bindings, no libsqlite, no external runtime libraries required.

🔁

Raft Consensus

Full leader election, log replication, crash recovery, and follower-to-leader write forwarding.

🌐

DNS Peer Discovery

Point a DNS A record at your cluster nodes. Scaling is a DNS update away.

🛢️

SQL Engine

Recursive-descent parser. JOINs, GROUP BY, subqueries, upserts with excluded.*, secondary indexes, transactions, savepoints.

📦

crystal-db API

Passes DB::DriverSpecs — the same compatibility suite the official crystal-sqlite3 driver runs. Connection pools, prepared statements, typed queries.

Embedded use

# shard.yml
dependencies:
  trash-panda-db:
    github: dirless/trash-panda-db
require "trash-panda-db"

DB.open("trashpanda:/var/lib/myapp/data.tpdb") do |db|
  db.exec "CREATE TABLE IF NOT EXISTS events (id INTEGER PRIMARY KEY, payload TEXT)"
  db.exec "INSERT INTO events VALUES (?, ?)", 1, "hello"
  row = db.query_one "SELECT payload FROM events WHERE id = ?", 1, as: String
end

Replicated cluster

# DNS A record: db-cluster.example.com → 10.0.0.1, 10.0.0.2, 10.0.0.3
# Run this on each node — peers discovered automatically from DNS

podman run -d \
  --name raft-node \
  -p 9002:9002 \
  trash-panda-raft \
  --raft   0.0.0.0:9001 \
  --client 0.0.0.0:9002 \
  --dns-peers db-cluster.example.com
# Write to any node — followers forward to the leader automatically
echo '{"action":"propose","sql":"INSERT INTO events VALUES (1,\"hello\")"}' \
  | nc -q1 10.0.0.2 9002

# Read from any node
echo '{"action":"local_query","sql":"SELECT * FROM events"}' \
  | nc -q1 10.0.0.3 9002

Install

# From source (requires Crystal ≥ 1.20 and just)
git clone https://github.com/dirless/trash-panda-db
cd trash-panda-db
just install   # builds --release and copies to /usr/local/bin
